# SlideDrop scale-up situation report

Created: 2026-05-01 18:43:54 UTC+8

## Summary

Responded to the expanded commercial-scale critique by shipping automation surfaces,
stronger conversion bookkeeping, CI and dependency hygiene improvements, and clearer
distribution docs.

## Changes shipped

- Headless `slidedrop convert` CLI plus localhost-only `slidedrop api` (`POST /v1/convert`).
- `.pptx` font preflight integrated into the GUI confirmation dialog.
- SHA-256 hashing on queue ingest, optional skip-if-unchanged using PDF sidecar metadata.
- LibreOffice per-job timeouts, readable-source retries, and `impress_pdf_Export` hooks
  for `.pptx` (including speaker-note toggle).
- Mock conversion engine moved to `tests/support` so production wheels omit test doubles.
- Single-version workflow: `scripts/windows/sync-version-info.py` regenerates Windows
  VERSIONINFO from `slidedrop.version`.
- CI: macOS build workflow, Pages deploy workflow, Dependabot, pytest coverage gate at
  65% with documented omissions for GUI/API entrypoints.
- Dependency pins in `requirements*.txt` / `pyproject.toml`; README notes hash-lock option.
- Inno Setup template, Linux headless notes, macOS `.icns` guidance asset readme.
- Optional Sentry wiring via `SLIDEDROP_SENTRY_DSN`.

## Explicit non-goals / follow-ups

- No bundled font payloads yet (preflight warns only).
- No SQLite resume queue or transactional batch recovery yet.
- No Authenticode / Apple notarization secrets wired (documented expectations only).
- No PDF geometric reflow beyond LibreOffice filter JSON passthrough.
- Branch protection itself is repository settings; documented in README.

## Validation

- `python -m pytest --cov=slidedrop --cov-fail-under=65`
